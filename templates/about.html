Personal Buffett

An AI-powered value investing assistant — built for someone who needed signal, not noise.

The Problem

My mum has been investing in A-shares for years. She reads financial news every morning, tracks eight stocks, and still finds it hard to know what actually matters on any given day. Most apps give her prices and charts — not judgment.

I built this for her. The goal was simple: each morning, she opens the app and gets one thing per stock — a clear, opinionated letter in plain language that tells her whether to hold, buy more, or start worrying.

What It Does

  • Daily pipeline — fetches real-time price, news, fund flow, and annual financials for every watchlist stock, then runs a Buffett-framework analysis using Groq's free Llama 3 70B API.
  • Buffett letter — each analysis is a first-person letter from "Warren Buffett (private edition)": moat direction, capital allocation quality, PE/PB vs 5-year history, fund flow, and a clear conclusion.
  • Wall Street-grade signals — ROIC trend, Owner Earnings, retained earnings efficiency (Buffett's capital allocation test), major-shareholder pledge ratio, margin balance direction, institutional holdings changes.
  • Prediction tracking — every buy/hold/reduce/sell prediction is logged. Seven and thirty days later, actual returns are back-filled automatically so the model's accuracy can be measured and improved.
  • Multi-market — A-shares via AKShare and Sina Finance, NZX and US stocks via yfinance.

What I Built

Data Layer

Full A-share pipeline: Sina hq API for real-time quotes, AKShare for financial abstracts (ROE, net margin, leverage, EPS), Sina balance sheet and cash flow APIs for ROIC and Owner Earnings calculation, plus pledge ratio, margin balance, and institutional holdings. NZ/US via yfinance.

Analysis Model

Designed the Buffett prompt framework from scratch: GREAT/GOOD/GRUESOME business classification, moat direction inference, capital allocation quality assessment, and loss-making company edge-case handling. Pre-compute trend directions in code before injection into the LLM prompt — this eliminates a class of "LLM confuses 12.6% > 8.6% with decline" errors.

Prediction System

Every analysis is treated as a falsifiable prediction. A daily launchd job back-fills actual 7d/30d returns and tracks accuracy by prediction type (buy/hold/reduce/sell). This is the feedback loop that lets the model improve.

Product Design

NYT-inspired newspaper layout. Watchlist sidebar with client-side sort, filter, and thumbnail mode. Stock fundamentals page with rating timeline, financial trend charts, and moat analysis. Multi-language (EN/ZH) with i18n JSON tables.

Infrastructure

macOS launchd for fully automated daily runs — pipeline at 08:30 CST, backfill at 17:00 NZT. Flask + SQLite backend designed to migrate to PostgreSQL without schema changes. Google OAuth + bcrypt password auth.

Tech Stack

Flask 3 SQLite AKShare yfinance Groq · Llama 3.3 70B Authlib · Google OAuth Bootstrap 5 Chart.js macOS launchd Python 3.11+

Design Principles

Three questions Warren Buffett asks about every business, hard-wired into every analysis prompt:

  1. Is this a great business? — GREAT (high ROE, low capital consumption) / GOOD (solid but needs reinvestment) / GRUESOME (grows fast, eats capital, doesn't earn)
  2. Is the moat widening or narrowing? — ROE and net margin trends, competitive signals in news, institutional flows
  3. Is the price reasonable? — PE/PB vs 5-year historical percentile, not just absolute numbers

The LLM doesn't replace judgment. It synthesises structured data into a readable letter that surfaces the most important signal for each stock, each day — so the human can focus on deciding, not filtering.
