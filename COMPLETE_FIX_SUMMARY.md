## 设计原则

### 数据流（从数据库到 LLM）
```
数据库（真实数据）
    ↓
Pipeline（不过滤）
    ↓
Buffett 分析器（格式化和分级）
    ├─ 红旗数据：单独分组、用图标标记、提前显示 🚨
    ├─ 关键指标：清晰表达、有单位、有范围参考
    └─ 新闻信号：量化指标（情绪值、影响力）
    ↓
LLM（完整数据 + 强制指导）
    ├─ Prompt 明确要求处理红旗
    ├─ Prompt 要求理由要基于数据
    └─ Prompt 要求好消息不能消除坏数据
    ↓
信（准确 + 有哲学 + 无忽视）
```

### 关键洞察
1. **数据准确性是基础**：没有真实数据，任何 prompt 都救不了
2. **格式可以强制 LLM 行为**：把重要数据优先显示、用图标标记，LLM 倾向于关注
3. **原则比硬规则有效**：用"对比是核心"比"每段必须有数字"更能激发 LLM 的思考
4. **红旗检查清单防止忽视**：明确列出需要提及的风险，LLM 就难以绕过
