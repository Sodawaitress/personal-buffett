# Beginner Investing Framework

给 Claude 和未来的我们看。

这份文档不是教人“炒股技巧”，而是定义 `Personal Buffett` 应该如何把复杂投资判断翻译成小白也能听懂的话。

目标只有一个：

`把数据 -> 解释 -> 判断 -> 风控` 这条链路讲明白。

---

## 1. 产品到底在做什么

`US-66` 已经完成了第一层：

- 把财务指标翻成人话
- 让用户知道 “ROE 20% 算不算好”
- 让用户知道 “PE 40 倍意味着市场在赌增长”

但一个真正能帮助非专业投资者做判断的产品，至少要有 4 层：

1. 数据层  
   看到 `PE / ROE / 净利率 / 负债率 / 现金流 / 价格位置`

2. 翻译层  
   告诉用户这些数字大概算 “好 / 一般 / 差”

3. 判断层  
   告诉用户：这家公司是怎么赚钱的，现在值不值得看，为什么

4. 风控层  
   告诉用户：如果这个判断错了，会错在哪里，什么时候该收手

当前产品最缺的是第 3 和第 4 层。

---

## 2. 小白真正需要的不是数据，而是判断骨架

对于非专业投资者，绝大多数时候不需要看到更多数据，而需要更稳定的“判断骨架”。

每只股票都应该尽量回答这 6 个问题：

1. 这家公司靠什么赚钱？
2. 它最重要的优势是什么？
3. 它当前最大的风险是什么？
4. 这个价格算便宜、合理还是贵？
5. 什么情况说明这次判断错了？
6. 今天更适合 `买入 / 持有 / 观察 / 回避` 哪一种动作？

如果这 6 个问题答不出来，说明产品还停留在“会展示数据”，还没有真正进入“帮助判断”。

---

## 3. 行业认知：不是知道很多，而是知道该看什么

### 3.1 小白版定义

行业认知，不是成为行业专家。

对产品来说，行业认知的目标只有 3 件事：

1. 知道这类公司通常靠什么赚钱
2. 知道这类公司最怕什么
3. 知道判断这类公司时，哪些指标最重要，哪些指标容易误导

一句话：

`行业认知 = 知道不同类型公司要用不同尺子去看`

### 3.2 为什么它重要

同样一个低 PE：

- 放在银行股身上，可能只是正常
- 放在周期股身上，可能是高点陷阱
- 放在成长股身上，可能说明增长已经坏掉
- 放在 ST / 事件股身上，可能根本没有意义

所以产品不能默认：

`一个指标，在所有公司上都代表同一个意思`

### 3.3 最小可用行业框架

对小白产品，不需要一开始做 30 个行业。

先把公司分成几类就够了：

1. 稳定价值股  
   例：白酒、家电、消费龙头、部分设备龙头  
   核心看：ROE、净利率、现金流、估值历史位置

2. 成长股  
   例：软件、半导体、平台型公司、部分医药创新  
   核心看：收入增速、毛利率、研发效率、增长是否在放缓

3. 周期股  
   例：钢铁、煤炭、化工、地产链、航运  
   核心看：行业景气位置、价格周期、现金流能不能穿越低谷

4. 金融股  
   例：银行、保险、券商  
   核心看：资产质量、资本充足性、盈利结构  
   不适合照搬普通工业股的 ROE/负债逻辑

5. 公用事业 / 高股息  
   例：电力、水务、燃气、运营型资产  
   核心看：分红可持续性、债务可持续性、监管风险

6. 事件驱动 / ST / 重整股  
   核心看：事件是否兑现、时间窗口、稀释风险、退市风险  
   不应该硬套巴菲特框架

这和产品现有的 `framework routing` 是一致的。

## 4. 估值框架：不是便宜不便宜，而是“对谁来说便宜”

### 4.1 小白版定义

估值框架不是：

- “PE 低就好”
- “PB 低就便宜”
- “跌很多了就安全”

估值框架真正要回答的是：

`对这种类型的公司来说，现在这个价格算不算值得付这个钱`

### 4.2 小白最常见的误区

1. 只看绝对 PE，不看业务类型
2. 只看现在便不便宜，不看过去是不是更便宜
3. 只看价格跌没跌，不看盈利有没有坏
4. 用同一套尺子看银行、白酒、科技、周期

### 4.3 产品里应该怎么讲

#### A. 稳定价值股

重点问：

- 这家公司长期赚不赚钱？
- 现在的价格，相对它历史上贵不贵？
- 现金流有没有跟上利润？

适合的表达：

- “公司赚钱能力还稳，当前估值低于它过去 5 年的大多数时间”
- “质量还行，但价格已经不便宜了，适合等更好位置”

#### B. 成长股

重点问：

- 增长还在不在？
- 毛利率有没有恶化？
- 当前估值是不是已经把未来几年都提前透支了？

适合的表达：

- “市场不是在给它现在定价，而是在给它未来几年的增长定价”
- “如果增速掉下来，现在这个价格就会显得贵”

#### C. 周期股

重点问：

- 现在是不是行业景气高点？
- 低估是便宜，还是高点利润假装出来的便宜？

适合的表达：

- “看起来 PE 很低，但可能只是因为行业正处在高景气”
- “这种股票不能只看便宜，还要看周期位置”

#### D. 事件驱动股

重点问：

- 事件是否真实推进？
- 时间、稀释、监管、退市这些风险谁更大？

适合的表达：

- “这不是按正常盈利估值的股票，而是在赌事件能不能兑现”

### 4.4 一个统一原则

对所有框架都成立的一句话：

`估值不是孤立数字，而是价格和质量、增长、风险三者之间的交换`

## 5. 风控逻辑：不是害怕，而是提前定义“错了怎么办”

### 5.1 小白版定义

风控不是“胆小”，也不是“看技术图吓自己”。

风控的本质是：

`在做决定之前，就先写好自己什么情况下算看错`

### 5.2 产品应该帮助用户回答的风控问题

1. 我这次是因为什么买 / 持有 / 观察？
2. 哪个条件一旦发生，就说明原逻辑被破坏了？
3. 这类票最多应该占我组合多少？
4. 现在错的概率大还是回报空间大？

### 5.3 最适合小白的 4 条风控原则

1. 不因为“跌很多了”就默认便宜
2. 不因为“别人都在说”就买
3. 买前先写错因条件
4. 单只股票不要一次压太大仓位

### 5.4 错因条件举例

#### 稳定价值股

- ROE 连续下滑
- 净利率明显恶化
- 现金流明显弱于利润
- 估值修复没来，反而基本面先坏

#### 成长股

- 收入增速大幅放缓
- 毛利率掉得很快
- 研发投入没有转成竞争力
- 市场份额没有继续扩大

#### 周期股

- 商品价格转头
- 行业供给扩张
- 利润高点不可持续

#### 事件股

- 核心事件延期 / 失败
- 稀释超预期
- 监管或法律风险放大

## 6. 对小白，产品输出必须尽量长成这样

不要只说：

- “ROE 高”
- “估值合理”
- “建议持有”

应该尽量长成：

1. 公司是怎么赚钱的
2. 最强优势是什么
3. 当前最大风险是什么
4. 当前价格处在什么位置
5. 什么情况下本次判断失效
6. 今天更适合做什么动作

### 6.1 一个推荐模板

#### 一句话结论

- 适合继续看 / 可以小仓位试 / 先别碰 / 只适合事件观察

#### 为什么

- 赚钱能力
- 护城河 / 增长 / 周期位置
- 估值位置

#### 风险

- 最重要的 1-2 个风险，不要列一堆

#### 失效条件

- 明确写出触发条件

#### 当前动作

- 买入 / 持有 / 观察 / 回避

---

## 7. 语言要求：一定要像解释给家人听

产品最终语言必须遵守这些原则：

1. 先说结论，再说原因
2. 尽量少用术语
3. 一个段落只讲一个意思
4. 不要堆很多指标，要解释“哪个最重要”
5. 不说假确定性
6. 风险要说人话，不要写成审计报告

### 7.1 好的表达

- “这家公司赚钱能力不错，但现在不是特别便宜。”
- “这只股票看起来便宜，但更像是行业高点带来的假便宜。”
- “如果后面利润继续下滑，这次‘买入’判断就站不住了。”

### 7.2 不好的表达

- “ROE 23%，处于优秀分位，建议积极关注。”
- “估值中枢回归逻辑具备一定成立条件。”
- “风险主要体现为阶段性经营波动及外部扰动。”

---

## 8. 给 Claude 的执行原则

Claude 在分析股票、设计文案、修改 prompt、规划 UI 时，应默认遵守下面 5 条：

1. 不把“更多数据”误当成“更多帮助”
2. 默认用户是非专业投资者
3. 先做解释，再做判断，再做风控
4. 不用同一套框架分析所有股票
5. 永远明确 “为什么成立” 和 “什么时候失效”

---

## 9. 一句话总纲

`Personal Buffett` 不应该只是一个会查数据、会打分、会写信的工具。

它真正要做的是：

`把复杂投资判断，翻译成非专业投资者也能理解、也能执行的结论。`
