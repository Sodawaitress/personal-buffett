## 🚨 最常见的错误模式（现在能自动检测）

### ❌ Bug Pattern 1: 变量名混淆

```javascript
// 错误代码（已自动检测）
const items = await fetch(...).json();
if (data && data.loading) {  // ❌ items 和 data 混淆了！
```

**自动检测**: ✅ `test_search_variable_consistency()` 会捕获这个

### ❌ Bug Pattern 2: 文件不同步

```javascript
// index.html
const data = await resp.json();
if (data.loading) { ... }  // 这是对的

// watchlist.html
const items = await resp.json();
if (data.loading) { ... }  // ❌ 这变成错的了！
```

**自动检测**: ✅ `test_search_variable_consistency()` 会捕获这个

### ❌ Bug Pattern 3: 数据类型不匹配

```javascript
const items = await fetch(...).json();
if (!Array.isArray(items) || !items.length) {
  // 当 items = {loading: true} 时，这里会错误触发
}
```

**自动检测**: ✅ `test_search_returns_list()` 会验证返回类型

## 🔍 故障排除

### Q: 测试显示"Query 'AAPL' 返回空结果"怎么办？
A: 这通常是临时现象（A股数据加载中或网络问题）。再运行一次测试。

### Q: 我修改了搜索代码但测试仍然通过，是否代表没有 bug？
A: 不一定。自动化测试只检测常见模式。还需要：
1. 手工测试（重要！）
2. 在浏览器中实际搜索
3. 清除缓存后重试

### Q: 如果两个文件的搜索代码不完全相同怎么办？
A: 当前 Checklist 能检测到不一致。长期目标是提取到 `static/search.js`，完全统一。
