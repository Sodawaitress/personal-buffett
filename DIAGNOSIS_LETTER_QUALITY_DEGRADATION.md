**期望流程**：

**问题**：
1. `fundamentals_str` 对非A股是否被正确格式化？
2. 是否包含了 ROE、利润率、债务比？
3. 数据格式是否足够清晰让 LLM 理解？

我们升级了 prompt，强制要求"有数据就用，没有就说数据不足"：

**问题**：

- ❌ 说明 LLM 根本没有收到这些数据，或者理解能力不足

## 具体诊断 - Intel 分析

### 数据应该有但没出现

**实际 Intel 数据**：
```
PE: None (未公开)
PB: 2.70
利润率: -0.505% ← 亏损！
ROE: 0.022% ← 接近零！
52周: $17.67-$62.08（当前 $61.72 = 99%高位）
```

**LLM 编造的数据**：
```
"年利润率为15%" ← 完全错了
"市占率稳定在20%左右" ← 没有这个字段
"历史平均PE为18.5" ← 编造的
```

### 为什么会编造？

**LLM 的思维过程**：
1. System Prompt 说："必须给出 GREAT/GOOD/GRUESOME"
2. LLM 看用户消息，缺少历史PE数据
3. LLM 没有收到"数据不足"的许可
4. LLM 的自我修正机制说："我应该能理解金融数据"
5. **LLM 开始幻觉/编造数据来"补全"答案**

这是 **LLM 的通病**，特别是在：
- Prompt 要求很强硬时
- 输入数据格式不清晰时
- 模型能力有限时（Groq 不如 GPT-4）

## 根本设计缺陷

**设计意图**（来自 `buffett_model.md`）：
- 用巴菲特框架分析股票
- 给出 GREAT/GOOD/GRUESOME 分类
- 基于**真实数据**做出投资结论

**实现缺陷**：
- 没有数据可用性保证机制
- LLM 可以随意编造数据
- 没有"数据验证"或"数据补全"的逻辑

**应该添加的**：
1. **数据不足时的备选分析路径**
2. **LLM 生成的数字验证**
3. **明确的"数据来源"标注**

## 建议

这个问题的根本是：
> **系统给了 LLM 太强硬的要求，但没给它足够的数据，所以它编造了。**

修复应该是：
1. ✅ 首先确保数据完整传递
2. ✅ 其次明确告诉 LLM 哪些数据是空的
3. ✅ 第三提供"缺数据时的分析策略"
4. ❓ 考虑换更好的 LLM（GPT-4 不会这么爱编造）
